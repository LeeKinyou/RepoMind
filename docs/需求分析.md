# 需求分析（Requirements Analysis）

## 1. 系统架构概览

```
                  +-------------------------------------------------+
                  |                  RepoMind CLI                   |
                  +-------------------------------------------------+

                  v                         v                         v
       +--------------------+     +--------------------+     +--------------------+
```

## 2. 系统功能性需求（FR）

### FR-1: 多级代码仓库索引构建（Repository Indexing）

**FR-1.1**：系统必须支持本地物理目录及远程 Git 仓库的自动检出与克隆。

**FR-1.2**：系统必须在不牺牲速度的前提下，快速建立文件、类（Class）、函数（Function）、模块依赖（Imports）及物理调用链（Calls）的静态关系图谱。

**FR-1.3**：支持对所有代码实体（函数体、文档字符串）进行段落切片并调用本地 Embedding 引擎（如 jina-code）进行向量编码持久化存储。

**FR-1.4**：支持 Python Type Hints 和类型推断，在无显式类型声明的情况下通过多策略级联算法推断变量和函数返回值类型。

### FR-2: 混合检索与图增强（Hybrid GraphRAG）

**FR-2.1**：提供双向检索接口：
- 支持传统的 BM25 精确符号名检索
- 基于代码向量库的隐式业务概念语义检索

**FR-2.2**：实现图拓扑向外扩散检索（Graph Expansion）：
- 在命中的种子节点周围自动向外 BFS 检索 2 圈（2-hop neighbors）
- 将其引用的上下文骨架（Skeletons）自动打包召回

### FR-3: 智能根因调查与自愈环（Root Cause Investigation & Self-Refine）

**FR-3.1**：接收包含 Exception Stack Trace + Issue Body 的故障快照输入。

**FR-3.2**：通过正则提取崩溃帧的物理文件名及精确报错行数，与静态索引图谱双向对齐，锁定 Proximate Cause。

**FR-3.3**：自愈自省循环（Self-Refining Sandbox）：
- 基于 LATS（Tree Search 蒙特卡洛树搜索）驱动 TestAgent 在隔离沙箱中自动编写 Reproducer 单元测试
- 确认复现错误，随后将错误堆栈反馈给 Solver 生成代码修复 Diff 补丁
- 最终沙箱回归测试通过方可输出 RCA 报告

### FR-4: Mermaid 架构可视化呈现（Architecture Mapping）

**FR-4.1**：提取局部 NetworkX 邻接子图，自动编译为结构化的 Mermaid Markdown 脚本。

**FR-4.2**：前端自动调用 mermaid.js 提供高颜值的 2D 调用流或依赖结构可视化拖拽面板。

## 3. 系统非功能性需求（NFR）

### NFR-1: 高性能本地极速解析（Performance）

**NFR-1.1**：对于 10 万行级别的中型项目，全量语法解析与关系建图（SQLite）耗时必须控制在 **15 秒以内**。

**NFR-1.2**：每次增量更新（File Watcher 触发内容哈希重构）耗时必须在 **2 秒以内**完成。

### NFR-2: 极低运行开销与 Token 优化（Cost Efficiency）

**NFR-2.1**：在单次架构理解或影响分析查询中，相比于传统的 Blind File Crawling（单次消耗约 41.2 万 Tokens），RepoMind 通过图拓扑检索和 AST 骨架化剪枝，单次会话 Token 消耗必须低于 **5,000 Tokens**，实现 **90% 以上的费用节省**。

**NFR-2.2**：单次图谱分析接口本地响应耗时（无 LLM 调用时）必须在 **5 毫秒以下**。

### NFR-3: 宿主机安全防护（Security & Containment）

**NFR-3.1**：严防 Agent 恶意命令执行或被污染仓库的 Prompt 注入攻击。

**NFR-3.2**：所有涉及 WritePatch 及 pytest/Shell 测试执行的动作，必须在隔离的容器（如 Docker 或基于 gVisor 的轻量化 Ephemeral 运行沙箱）中运行，绝不允许污染开发者宿主机物理文件系统。

### NFR-4: 零配置与完全离线工作能力（Local-first）

系统需打包为单文件二进制或轻量化 Python 脚手架，内置嵌入式 SQLite 与本地 LanceDB 向量存储，支持在无网络环境下通过本地模型（如 Ollama）正常执行索引、检索与可视化渲染。

## 4. MVP 裁剪边界（MVP Scope Boundary）

为了确保 1 周内绝对可以跑通、真实运行并向技术面试官展示，系统做出了极其务实的工程边界划定：

### 4.1 必须包含的核心功能 (MVP In-Scope)

| 功能模块 | 具体内容 |
|---------|---------|
| 语法解析 | Tree-sitter + Jedi 双引擎 Python 解析（首期仅支持 Python） |
| 类型绑定 | 渐进式类型推断算法（Type Hints + 赋值推断 + 鸭子类型匹配） |
| 图存储 | SQLite + NetworkX 内存混合检索（单文件存储） |
| 故障定位 | 基于正则解析的 Exception Trace 逆向对齐映射 |
| 流程编排 | 硬编码 Linear State Machine 管道流编排 |
| 可视化 | 基于 Mermaid.js 的局部拓扑图轻量级 Web 渲染 |
| 自愈验证 | 沙箱测试执行自愈环（支持 pytest 自动验证） |

### 4.2 坚决砍掉的非必要模块 (Out of Scope)

| 功能模块 | 排除原因 |
|---------|---------|
| 多语言支持 | 全语种支持（Go/Rust/C++ 等）留待二期扩展 |
| 重型类型检查 | 完整的 IDE 级别类型检查与编译期检查过于复杂 |
| 生产级数据库 | Neo4j 生产级数据库常驻进程安装资源消耗大 |
| 实时流量接入 | 接入 OpenTelemetry、Dynatrace 生产实时流量 |
| 自由协商架构 | 多 Agent 闲聊型自由协商、冲突达成共识架构 |
| 高级渲染引擎 | 自研高性能 D3.js 二维空间受力向导渲染引擎 |
| 集群调度 | 跨主机的远程多点 K8s 集群测试资源调度 |

## 5. 推荐技术栈

| 技术维度 | 推荐组件 | 选型理由 |
|---------|---------|---------|
| 基础语言环境 | Python 3.13 | 保证对 AI 系统集成库、NetworkX 和 AST 解析的最佳生态兼容 |
| 词法与语法分析 | py-tree-sitter (Python vendor) | 纳秒级极速解析，拥有绝佳的语法容错能力 |
| 类型推断引擎 | Jedi | Python 静态分析库，支持类型推断、跳转定义、自动补全 |
| 关系图持久化存储 | SQLite 3 + GraphQLite 扩展 | 零配置单文件，极佳的本地可移植性，原生支持 Cypher 检索 |
| 向量存储引擎 | LanceDB (Disk-based) | 无服务器架构，磁盘存储无内存占用，支持极其轻量和极速的向量混合检索 |
| 内存图论计算 | NetworkX | 高度成熟的经典图计算算法（PageRank, Louvain），冷启动加载秒级完成 |
| 大模型接口封装 | LiteLLM | 一行代码自由切换本地 DeepSeek 蒸馏模型或云端 Claude Sonnet 4.6 |
| CLI 命令行构建 | Typer | 极速生成生产级的终端控制台交互界面 |
| 测试框架 | pytest | Python 标准测试框架，支持参数化、fixtures、插件扩展 |

## 6. 1 周敏捷开发排期（7日落地实操线）

| 开发周期 | 核心里程碑 | 每日具体研发任务与可运行产出 |
|---------|-----------|---------------------------|
| Day 1 | Tree-sitter 本地建库 | 编写 Python 语法对应的 Tree-sitter 词法解析，提取类、函数签名、物理文件范围，设计 SQLite 静态边表，输出基础索引 CLI |
| Day 2 | 渐进式类型推断构建 | 实现 6 策略级联类型推断算法（Type Hints + 赋值推断 + 鸭子类型）；集成 LanceDB 向量提取函数体 Embedding，跑通基于 jina-code 的方法级语义检索 |
| Day 3 | NetworkX 映射与 Mermaid 渲染 | 构建 SQLite 到 NetworkX 内存图的 pipeline，计算 PageRank 枢纽度；根据 NetworkX 局部跳转子图，自动输出规范的 Mermaid Markdown 依赖文本 |
| Day 4 | Exception Trace 逆向对齐与 RCA | 编写正则解析器抽取报错帧中的类与物理行；利用静态图 BFS 扩展机制向外抓取 2 跳代码上下文送 LiteLLM，一秒生成 RCA 报告 |
| Day 5 | TestAgent 沙箱自愈验证 | 编写 TestAgent 驱动 LLM 自动生成 pytest 用例，在本地隔离的沙箱（Docker）内运行；抓取 traceback 报错进行 patch 自愈重试 |
| Day 6 | RCEGen 裁判评测与基准运行 | 编写基于双大模型裁判打分的 RCEGen 极简评估脚本（评估 Correctness, Clarity, Depth）；导入 10 个公开的 Verified 样本，收集 Indexing Time, Recall@5 和 RCA Accuracy 指标 |
| Day 7 | 控制台精装与面试整备 | 使用 Typer 润色控制台输出（彩色面板、自愈日志流），录制高可信的功能演示 GIF，背诵简历技术细节 |

## 7. 验收标准

### 7.1 性能验收标准

- [ ] 10 万行项目全量索引 ≤ 15 秒
- [ ] 增量更新 ≤ 2 秒
- [ ] 单次查询 Token 消耗 ≤ 5,000
- [ ] 图谱分析响应 ≤ 5 毫秒

### 7.2 功能验收标准

- [ ] 支持 Python 项目本地索引构建
- [ ] 支持 Python Type Hints 类型提取
- [ ] 支持渐进式类型推断（Type Hints + 赋值推断 + 鸭子类型）
- [ ] 支持混合检索（BM25 + 语义）
- [ ] 支持 2-hop 图拓扑扩展
- [ ] 支持 Exception Trace 逆向对齐
- [ ] 支持 Mermaid 可视化输出
- [ ] 支持沙箱自愈验证

### 7.3 质量验收标准

- [ ] 类型推断准确率 ≥ 70%（无 Type Hints 场景）
- [ ] 调用图拓扑精度 ≥ 85%（有 Type Hints 场景 ≥ 95%）
- [ ] 检索 Recall@5 显著优于纯向量检索
- [ ] RCA 报告准确性通过 RCEGen 评测
- [ ] 所有测试在隔离沙箱中执行
