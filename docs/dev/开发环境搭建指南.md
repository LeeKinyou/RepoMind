# 开发环境搭建指南

## 1. 系统要求

| 要求 | 最低版本 | 推荐版本 | 说明 |
|------|---------|---------|------|
| 操作系统 | Windows 10 / macOS 12 / Ubuntu 20.04 | 最新稳定版 | 全平台支持 |
| Python | 3.13 | 3.13 | 必须 3.13，使用了 `match` 语句等新特性 |
| Git | 2.30 | 最新版 | 用于仓库克隆和版本管理 |
| Docker（可选） | 20.10 | 最新版 | 用于沙箱验证功能 |
| 内存 | 8 GB | 16 GB | NetworkX 图计算需要足够内存 |
| 磁盘 | 1 GB 可用空间 | 5 GB | 大型项目索引需要更多空间 |

## 2. 环境准备

### 2.1 安装 Python 3.13

**Windows**：

```powershell
# 方式 1: 从官网下载安装
# 访问 https://www.python.org/downloads/ 下载 Python 3.13 安装包
# 安装时勾选 "Add Python to PATH"

# 方式 2: 使用 winget
winget install Python.Python.3.13

# 方式 3: 使用 pyenv-win
pyenv install 3.13.0
pyenv global 3.13.0
```

**macOS**：

```bash
# 使用 Homebrew
brew install python@3.13

# 或使用 pyenv
brew install pyenv
pyenv install 3.13.0
pyenv global 3.13.0
```

**Linux (Ubuntu/Debian)**：

```bash
# 使用 deadsnakes PPA
sudo add-apt-repository ppa:deadsnakes/ppa
sudo apt update
sudo apt install python3.13 python3.13-venv python3.13-dev
```

验证安装：

```bash
python --version
# Python 3.13.x
```

### 2.2 安装 Git

```bash
# Windows
winget install Git.Git

# macOS
brew install git

# Linux
sudo apt install git
```

### 2.3 安装 Docker（可选）

Docker 仅用于沙箱验证功能，MVP 开发阶段可以跳过。

```bash
# Windows: 安装 Docker Desktop
# https://www.docker.com/products/docker-desktop/

# macOS
brew install --cask docker

# Linux
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

## 3. 项目初始化

### 3.1 克隆仓库

```bash
git clone git@github.com:LeeKinyou/RepoMind.git
cd RepoMind
```

### 3.2 创建虚拟环境

```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows PowerShell
.venv\Scripts\Activate.ps1

# Windows CMD
.venv\Scripts\activate.bat

# macOS / Linux
source .venv/bin/activate
```

验证虚拟环境已激活（命令行前应出现 `(.venv)` 前缀）：

```bash
# 确认 Python 版本和路径
python --version   # Python 3.13.x
which python       # 应指向 .venv 目录

# 确认 uv 可用
uv --version
```

### 3.3 安装依赖

本项目使用 **uv** 作为包管理器（而非 pip）。

```bash
# 安装 uv（如果尚未安装）
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows PowerShell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 安装项目本身（开发模式）+ 所有依赖
uv pip install -e ".[dev]"

# 同步依赖（精确安装锁定版本）
uv pip sync requirements.txt
```

### 3.4 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env，填入你的 LLM API Key
# REPOMIND_LLM_API_KEY=sk-xxx
```

所有系统级配置和密钥都通过 `.env` 文件注入，`.env` 已在 `.gitignore` 中不会被提交。

## 5. 开发工具配置

### 5.1 VS Code 配置（推荐）

在项目根目录创建 `.vscode/settings.json`：

```json
{
    "python.defaultInterpreterPath": "${workspaceFolder}/.venv/bin/python",
    "python.terminal.activateEnvironment": true,
    "[python]": {
        "editor.defaultFormatter": "charliermarsh.ruff",
        "editor.formatOnSave": true,
        "editor.codeActionsOnSave": {
            "source.fixAll": "explicit",
            "source.organizeImports": "explicit"
        }
    },
    "python.testing.pytestEnabled": true,
    "python.testing.pytestArgs": [
        "tests"
    ],
    "files.exclude": {
        "**/__pycache__": true,
        "**/.pytest_cache": true,
        "**/.mypy_cache": true,
        ".repomind": true
    }
}
```

推荐安装的 VS Code 扩展：

- **Python** (ms-python.python) — Python 语言支持
- **Pylance** (ms-python.vscode-pylance) — 类型检查与智能提示
- **Ruff** (charliermarsh.ruff) — 代码格式化
- **Even Better TOML** (tamasfe.even-better-toml) — pyproject.toml 支持

### 5.2 PyCharm 配置

1. 打开项目目录
2. `Settings → Project → Python Interpreter → Add Interpreter → Existing Environment`
3. 选择 `.venv/bin/python`（或 Windows 上的 `.venv\Scripts\python.exe`）
4. `Settings → Tools → Python Integrated Tools → Testing → Default test runner → pytest`

## 6. 代码规范

### 6.1 格式化与 Lint

项目使用 **Ruff** 进行代码格式化和静态检查：

```bash
# 格式化代码
uv run ruff format src/ tests/

# 检查代码问题
uv run ruff check src/ tests/

# 自动修复可修复的问题
uv run ruff check --fix src/ tests/
```

### 6.2 类型检查

```bash
uv run mypy src/repomind/ --ignore-missing-imports
```

### 6.3 提交规范

Commit Message 格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

类型（type）：

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档变更 |
| `style` | 代码格式（不影响逻辑） |
| `refactor` | 重构（既不是新功能也不是修复） |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具变更 |

示例：

```
feat(parser): add Tree-sitter Python parser integration

- Implement TreeSitterParser class for Python AST extraction
- Add symbol extraction for classes, functions, imports
- Support incremental parsing with content hash tracking

Closes #12
```

## 7. 运行测试

### 7.1 运行全部测试

```bash
# 运行所有测试
uv run pytest

# 运行并显示覆盖率
uv run pytest --cov=src/repomind --cov-report=term-missing

# 运行并生成 HTML 覆盖率报告
uv run pytest --cov=src/repomind --cov-report=html
```

### 7.2 运行特定测试

```bash
# 运行单个测试文件
uv run pytest tests/unit/test_parser.py

# 运行单个测试函数
uv run pytest tests/unit/test_parser.py::test_parse_python_file

# 运行匹配关键字的测试
uv run pytest -k "parser"

# 运行标记为 slow 的测试
uv run pytest -m slow
```

### 7.3 测试目录结构

```
tests/
├── unit/                       # 单元测试
│   ├── test_parser.py          # 解析引擎测试
│   ├── test_type_inference.py  # 类型推断测试
│   ├── test_call_graph.py      # 调用图测试
│   └── test_retrieval.py       # 检索引擎测试
├── integration/                # 集成测试
│   ├── test_index_service.py   # 索引服务集成测试
│   ├── test_query_service.py   # 查询服务集成测试
│   └── test_rca_service.py     # RCA 服务集成测试
├── fixtures/                   # 测试数据
│   ├── sample_project/         # 示例 Python 项目
│   ├── sample_traces/          # 示例 Stack Trace
│   └── expected_outputs/       # 期望输出
├── conftest.py                 # pytest 全局配置
└── README.md                   # 测试说明
```

## 8. 本地模型配置（可选）

如果希望完全离线运行，可以通过 Ollama 本地模型替代云端 API：

### 8.1 安装 Ollama

```bash
# macOS / Linux
curl -fsSL https://ollama.ai/install.sh | sh

# Windows: 从 https://ollama.ai 下载安装包
```

### 8.2 下载模型

```bash
# 下载 DeepSeek 蒸馏模型（推荐，速度快）
ollama pull deepseek-coder:6.7b

# 或下载更大的模型（质量更高）
ollama pull codellama:13b
```

### 8.3 配置 RepoMind 使用本地模型

在 `.env` 文件中修改以下配置：

```bash
REPOMIND_LLM_PROVIDER=ollama
REPOMIND_LLM_MODEL=deepseek-coder:6.7b
REPOMIND_LLM_BASE_URL=http://localhost:11434
```

## 9. 常见问题

### 9.1 tree-sitter 安装失败

**问题**：`uv pip install tree-sitter` 编译报错

**解决**：

```bash
# Windows: 安装 Visual C++ Build Tools
# https://visualstudio.microsoft.com/visual-cpp-build-tools/

# Linux: 安装编译工具
sudo apt install build-essential

# macOS
xcode-select --install
```

### 9.2 LanceDB 安装失败

**问题**：LanceDB 依赖 Rust 编译工具链

**解决**：

```bash
# 安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
uv pip install lancedb
```

### 9.3 内存不足

**问题**：大型项目索引时 OOM

**解决**：

在 `.env` 中添加：

```bash
REPOMIND_MAX_GRAPH_NODES=100000
```

### 9.4 Docker 沙箱不可用

**问题**：沙箱测试功能无法使用

**解决**：

```bash
# 检查 Docker 是否运行
docker info

# 如果 Docker 不可用，在 .env 中切换为子进程模式（安全性降低）
# REPOMIND_SANDBOX_MODE=subprocess
```

## 10. 开发工作流

### 10.1 典型开发流程

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 创建功能分支
git checkout -b feat/my-feature

# 3. 开发 + 测试
# ... 编写代码 ...
uv run pytest tests/unit/

# 4. 格式化 + Lint
uv run ruff format src/ tests/
uv run ruff check src/ tests/

# 5. 提交
git add .
git commit -m "feat(parser): add my feature"

# 6. 推送并创建 PR
git push origin feat/my-feature
```

### 10.2 日常用到的命令速查

```bash
# 激活虚拟环境
.venv\Scripts\Activate.ps1        # Windows PowerShell
source .venv/bin/activate          # macOS / Linux

# 安装依赖
uv pip install -e ".[dev]"

# 运行测试
uv run pytest
uv run pytest --cov=src/repomind

# 代码质量
uv run ruff format src/ tests/
uv run ruff check --fix src/ tests/
uv run mypy src/repomind/

# 运行 RepoMind CLI
uv run repomind index ./tests/fixtures/sample_project
uv run repomind query "some question"
uv run repomind stats
```
