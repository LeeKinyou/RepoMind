# 数据库设计文档

## 1. 存储架构概览

RepoMind 采用三层存储架构：

| 层级 | 技术 | 用途 | 特点 |
|------|------|------|------|
| 结构存储 | SQLite | 符号、关系、元数据 | 零配置、单文件、SQL 查询 |
| 向量存储 | LanceDB | 代码 Embedding | 磁盘存储、无服务器、混合检索 |
| 内存图 | NetworkX | 图算法计算 | 微秒级查询、PageRank/BFS/Louvain |

```
┌─────────────────────────────────────────────────────────────┐
│                    Storage Architecture                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐     ┌─────────────┐     ┌─────────────┐  │
│   │   SQLite    │     │   LanceDB   │     │  NetworkX   │  │
│   │ (Structure) │     │  (Vectors)  │     │  (In-Memory)│  │
│   └──────┬──────┘     └──────┬──────┘     └──────┬──────┘  │
│          │                   │                   │          │
│          v                   v                   v          │
│   ┌──────┴──────┐     ┌──────┴──────┐     ┌──────┴──────┐  │
│   │  - Files    │     │  - Method   │     │  - Call     │  │
│   │  - Classes  │     │    Embeddings│    │    Graph    │  │
│   │  - Methods  │     │  - Doc      │     │  - Import   │  │
│   │  - Imports  │     │    Strings  │     │    Graph    │  │
│   │  - Calls    │     │             │     │  - Inherit  │  │
│   └─────────────┘     └─────────────┘     │    Graph    │  │
│                                           └─────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 2. SQLite 表结构设计

### 2.1 文件表 (files)

存储被索引的源代码文件元数据。

```sql
CREATE TABLE files (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    path            TEXT    NOT NULL UNIQUE,       -- 文件相对路径
    hash            TEXT    NOT NULL,               -- 文件内容 SHA-256 哈希
    language        TEXT    NOT NULL,               -- 语言标识 (python)
    line_count      INTEGER NOT NULL DEFAULT 0,     -- 文件行数
    size_bytes      INTEGER NOT NULL DEFAULT 0,     -- 文件大小（字节）
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_files_path ON files(path);
CREATE INDEX idx_files_hash ON files(hash);
CREATE INDEX idx_files_language ON files(language);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO | 自增主键 |
| `path` | TEXT | NOT NULL, UNIQUE | 文件相对路径，作为唯一标识 |
| `hash` | TEXT | NOT NULL | 内容哈希，用于增量更新判断 |
| `language` | TEXT | NOT NULL | 语言标识，支持未来多语言扩展 |
| `line_count` | INTEGER | DEFAULT 0 | 文件行数，用于统计 |
| `size_bytes` | INTEGER | DEFAULT 0 | 文件大小，用于过滤大文件 |

### 2.2 符号表 (symbols)

存储类、函数、方法等代码符号。

```sql
CREATE TABLE symbols (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    file_id         INTEGER NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    name            TEXT    NOT NULL,               -- 符号名称 (如 "UserService")
    type            TEXT    NOT NULL,               -- 符号类型: class / function / method
    qualified_name  TEXT    NOT NULL,               -- 完整限定名 (如 "auth.service.UserService")
    start_line      INTEGER NOT NULL,               -- 起始行号
    end_line        INTEGER NOT NULL,               -- 结束行号
    docstring       TEXT,                           -- 文档字符串
    signature       TEXT,                           -- 函数签名 (如 "def login(self, user: str) -> bool")
    is_exported     BOOLEAN NOT NULL DEFAULT TRUE,  -- 是否被导出
    parent_class_id INTEGER REFERENCES symbols(id), -- 所属类 ID（方法才有）
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_symbols_file ON symbols(file_id);
CREATE INDEX idx_symbols_name ON symbols(name);
CREATE INDEX idx_symbols_qualified ON symbols(qualified_name);
CREATE INDEX idx_symbols_type ON symbols(type);
CREATE INDEX idx_symbols_parent_class ON symbols(parent_class_id);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO | 自增主键 |
| `file_id` | INTEGER | FK → files.id | 所属文件 |
| `name` | TEXT | NOT NULL | 短名称 |
| `type` | TEXT | NOT NULL | `class` / `function` / `method` |
| `qualified_name` | TEXT | NOT NULL | 完整限定名，全局唯一 |
| `start_line` | INTEGER | NOT NULL | 起始行号 |
| `end_line` | INTEGER | NOT NULL | 结束行号 |
| `docstring` | TEXT | NULLABLE | 文档字符串 |
| `signature` | TEXT | NULLABLE | 函数签名 |
| `is_exported` | BOOLEAN | DEFAULT TRUE | 是否公开导出 |
| `parent_class_id` | INTEGER | FK → symbols.id | 方法所属的类 |

**符号类型枚举**：

| type 值 | 说明 | 示例 |
|---------|------|------|
| `class` | 类定义 | `class UserService:` |
| `function` | 顶层函数 | `def helper():` |
| `method` | 类方法 | `def UserService.login(self):` |

### 2.3 类型信息表 (type_info)

存储渐进式类型推断的结果。

```sql
CREATE TABLE type_info (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol_id           INTEGER NOT NULL REFERENCES symbols(id) ON DELETE CASCADE,
    parameter_name      TEXT,                       -- 参数名（NULL 表示返回值）
    type_annotation     TEXT,                       -- 显式类型提示
    inferred_type       TEXT,                       -- 推断类型
    confidence          REAL    NOT NULL DEFAULT 0,  -- 置信度 (0.0 ~ 1.0)
    inference_strategy  TEXT    NOT NULL,            -- 推断策略
    evidence            TEXT,                       -- 推断证据（JSON）
    created_at          TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_type_info_symbol ON type_info(symbol_id);
CREATE INDEX idx_type_info_strategy ON type_info(inference_strategy);
CREATE INDEX idx_type_info_confidence ON type_info(confidence);
```

**推断策略枚举**：

| strategy 值 | 置信度 | 说明 |
|-------------|--------|------|
| `explicit_hint` | 0.95 | 显式 Type Hints |
| `import_mapping` | 0.90 | Import 映射绑定 |
| `self_inference` | 0.85 | self 参数类型推断 |
| `assignment` | 0.70 | 赋值语句推断 |
| `jedi` | 0.60 | Jedi 引擎推断 |
| `duck_typing` | 0.40 | 鸭子类型匹配 |

### 2.4 导入关系表 (imports)

存储文件间的 import 关系。

```sql
CREATE TABLE imports (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    file_id         INTEGER NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    module_path     TEXT    NOT NULL,               -- 模块路径 (如 "auth.service")
    imported_name   TEXT,                           -- 导入的符号名 (如 "UserService")
    alias           TEXT,                           -- 别名 (如 "from x import y as z" 中的 z)
    is_relative     BOOLEAN NOT NULL DEFAULT FALSE, -- 是否相对导入
    relative_level  INTEGER NOT NULL DEFAULT 0,     -- 相对导入层级 (from ..x 中的 2)
    line_number     INTEGER,                        -- 导入语句所在行号
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_imports_file ON imports(file_id);
CREATE INDEX idx_imports_module ON imports(module_path);
CREATE INDEX idx_imports_name ON imports(imported_name);
```

**示例数据**：

| file_id | module_path | imported_name | alias | is_relative | relative_level |
|---------|-------------|---------------|-------|-------------|----------------|
| 1 | auth.models | User | NULL | FALSE | 0 |
| 1 | ..utils | helper | h | TRUE | 2 |
| 2 | datetime | datetime | NULL | FALSE | 0 |

### 2.5 调用关系表 (calls)

存储函数/方法间的调用关系。

```sql
CREATE TABLE calls (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    caller_id       INTEGER NOT NULL REFERENCES symbols(id) ON DELETE CASCADE,
    callee_id       INTEGER NOT NULL REFERENCES symbols(id) ON DELETE CASCADE,
    call_type       TEXT    NOT NULL,               -- 调用类型
    confidence      REAL    NOT NULL DEFAULT 1.0,   -- 绑定置信度
    line_number     INTEGER,                        -- 调用语句所在行号
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_calls_caller ON calls(caller_id);
CREATE INDEX idx_calls_callee ON calls(callee_id);
CREATE INDEX idx_calls_type ON calls(call_type);
CREATE INDEX idx_calls_confidence ON calls(confidence);
```

**调用类型枚举**：

| call_type 值 | 置信度范围 | 说明 |
|-------------|-----------|------|
| `direct` | 0.90 ~ 1.0 | 直接函数调用 `func()` |
| `method` | 0.85 ~ 0.95 | 已知类型的对象方法调用 `obj.method()` |
| `self` | 0.95 ~ 1.0 | 类内 self 调用 `self.method()` |
| `duck` | 0.30 ~ 0.60 | 鸭子类型推断的调用 |

### 2.6 继承关系表 (inherits)

存储类之间的继承关系。

```sql
CREATE TABLE inherits (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    child_id        INTEGER NOT NULL REFERENCES symbols(id) ON DELETE CASCADE,
    parent_id       INTEGER REFERENCES symbols(id), -- 父类符号 ID（项目内定义时有值）
    parent_name     TEXT    NOT NULL,               -- 父类名称（可能是外部依赖）
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_inherits_child ON inherits(child_id);
CREATE INDEX idx_inherits_parent ON inherits(parent_id);
CREATE INDEX idx_inherits_parent_name ON inherits(parent_name);
```

## 4. LanceDB 向量存储设计

### 4.1 Schema 定义

```python
VECTOR_SCHEMA = {
    "symbol_id": "int64",           # 关联 symbols.id
    "symbol_name": "string",        # 符号名称
    "symbol_type": "string",        # class / function / method
    "qualified_name": "string",     # 完整限定名
    "file_path": "string",          # 文件路径
    "start_line": "int32",          # 起始行号
    "end_line": "int32",            # 结束行号
    "code_text": "string",          # 完整代码文本
    "docstring": "string",          # 文档字符串
    "signature": "string",          # 函数签名
    "embedding": "float32[768]",    # 嵌入向量 (jina-code-embeddings-v2)
    "created_at": "string"          # 创建时间 ISO 8601
}
```

### 4.2 索引配置

```python
INDEX_CONFIG = {
    "metric_type": "cosine",        # 余弦相似度
    "index_type": "IVF_FLAT",       # 倒排索引 + 平坦量化
    "nlist": 1024,                  # 聚类中心数
    "nprobe": 64                    # 查询时探测的聚类数
}
```

### 4.3 Embedding 模型

| 模型 | 维度 | 说明 |
|------|------|------|
| `jinaai/jina-embeddings-v2-base-code` | 768 | 默认，代码专用，支持多语言 |
| `sentence-transformers/all-MiniLM-L6-v2` | 384 | 轻量备选，速度快 |
| `BAAI/bge-base-en-v1.5` | 768 | 英文代码备选 |

### 4.4 嵌入文本构建策略

每个符号的嵌入输入文本格式：

```
{symbol_type}: {qualified_name}
Signature: {signature}
Docstring: {docstring}
Code: {code_text}
```

示例：

```
method: auth.service.UserService.authenticate
Signature: def authenticate(self, username: str, password: str) -> bool
Docstring: Authenticate user with username and password.
Code: def authenticate(self, username, password):
    user = self.repo.find_by_username(username)
    if not user:
        return False
    return verify_password(password, user.hashed_password)
```

## 5. NetworkX 内存图设计

### 5.1 图结构

```python
# 主调用图
call_graph = nx.DiGraph()

# 节点属性
call_graph.add_node(symbol_id, **{
    "name": "UserService",
    "type": "class",
    "qualified_name": "auth.service.UserService",
    "file_path": "auth/service.py",
    "start_line": 10,
    "end_line": 50,
    "page_rank": 0.0234,           # PageRank 中心度
    "community": 3,                # Louvain 社群编号
    "in_degree": 15,               # 入度（被调用次数）
    "out_degree": 8                # 出度（调用他人次数）
})

# 边属性
call_graph.add_edge(caller_id, callee_id, **{
    "type": "calls",               # calls / imports / inherits
    "confidence": 0.95,
    "line_number": 42
})
```

### 5.2 子图类型

| 子图 | 边类型 | 用途 |
|------|--------|------|
| `call_subgraph` | `calls` | 调用关系，用于理解执行流程 |
| `import_subgraph` | `imports` | 导入关系，用于分析模块边界 |
| `inherit_subgraph` | `inherits` | 继承关系，用于理解 OOP 结构 |
| `combined_graph` | 全部 | 综合图，用于全局分析 |

### 5.3 图算法

| 算法 | 用途 | 输出 |
|------|------|------|
| PageRank | 识别核心枢纽节点 | 每个节点的中心度分数 |
| Louvain | 社群聚类，识别高内聚模块 | 每个节点的社群编号 |
| BFS/DFS | 图拓扑扩展 | N 跳邻居节点集合 |
| 最短路径 | 两个符号间的调用路径 | 路径节点序列 |
| 连通分量 | 识别孤立模块 | 连通分量列表 |

## 6. 数据迁移策略

### 6.1 版本管理

索引格式版本通过 `index_metadata` 表的 `version` 键管理：

```sql
INSERT INTO index_metadata (key, value) VALUES ('version', '1.0.0');
```

### 6.2 迁移脚本

```python
MIGRATIONS = {
    "1.0.0": None,  # 初始版本，无需迁移
    "1.1.0": [
        "ALTER TABLE symbols ADD COLUMN parent_class_id INTEGER REFERENCES symbols(id);",
        "CREATE INDEX idx_symbols_parent_class ON symbols(parent_class_id);",
    ],
    "2.0.0": [
        # 未来版本的迁移 SQL
    ],
}

def migrate(db_path: str, from_version: str, to_version: str):
    """执行数据库迁移"""
    versions = sorted(MIGRATIONS.keys())
    start = versions.index(from_version) + 1
    end = versions.index(to_version) + 1

    with sqlite3.connect(db_path) as conn:
        for version in versions[start:end]:
            for sql in MIGRATIONS[version]:
                conn.execute(sql)
            conn.execute(
                "UPDATE index_metadata SET value = ? WHERE key = 'version'",
                (version,)
            )
            conn.commit()
```

### 6.3 兼容性策略

| 场景 | 处理方式 |
|------|---------|
| 版本向前兼容 | 自动执行迁移脚本 |
| 版本向后不兼容 | 提示用户删除旧索引并重建 |
| 迁移失败 | 回滚事务，保留旧索引，打印错误信息 |

## 7. 性能优化

### 7.1 SQLite 优化

```sql
-- 启用 WAL 模式（提升并发读性能）
PRAGMA journal_mode = WAL;

-- 设置缓存大小（页数，负值为 KB）
PRAGMA cache_size = -64000;  -- 64 MB

-- 启用内存映射
PRAGMA mmap_size = 268435456;  -- 256 MB

-- 同步模式（开发用 NORMAL，生产用 FULL）
PRAGMA synchronous = NORMAL;

-- 临时表存储在内存
PRAGMA temp_store = MEMORY;
```

### 7.2 查询优化

```sql
-- 高频查询：按名称查找符号
SELECT * FROM symbols WHERE name = ? AND type = ?;

-- 高频查询：获取调用链
SELECT s2.*
FROM calls c
JOIN symbols s1 ON c.caller_id = s1.id
JOIN symbols s2 ON c.callee_id = s2.id
WHERE s1.qualified_name = ?;

-- 高频查询：获取继承关系
SELECT s2.*
FROM inherits i
JOIN symbols s1 ON i.child_id = s1.id
JOIN symbols s2 ON i.parent_id = s2.id
WHERE s1.qualified_name = ?;
```

### 7.3 批量操作

```python
# 批量插入符号
cursor.executemany(
    "INSERT INTO symbols (file_id, name, type, qualified_name, start_line, end_line) VALUES (?, ?, ?, ?, ?, ?)",
    symbols_data
)

# 批量插入调用关系
cursor.executemany(
    "INSERT INTO calls (caller_id, callee_id, call_type, confidence, line_number) VALUES (?, ?, ?, ?, ?)",
    calls_data
)
```

### 8.3 数据导出

```python
import sqlite3
import json

def export_to_json(db_path: str, output_path: str):
    """将索引数据导出为 JSON"""
    with sqlite3.connect(db_path) as conn:
        conn.row_factory = sqlite3.Row
        data = {
            "symbols": [dict(row) for row in conn.execute("SELECT * FROM symbols")],
            "calls": [dict(row) for row in conn.execute("SELECT * FROM calls")],
            "imports": [dict(row) for row in conn.execute("SELECT * FROM imports")],
            "inherits": [dict(row) for row in conn.execute("SELECT * FROM inherits")],
        }
    with open(output_path, 'w') as f:
        json.dump(data, f, indent=2)
```
